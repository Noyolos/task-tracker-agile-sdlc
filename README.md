# Task Tracker Agile SDLC Project

## Project Overview

Task Tracker is a simple web-based task management application developed for the BAI21113 Software Engineering assignment. The application is designed for internal company use, allowing users to create, view, update, delete, and track tasks.

The project demonstrates the use of Agile SDLC, iterative and incremental development, and GitHub collaboration practices. The system was developed across two iterations. The first iteration focused on a minimum viable version, while the second iteration added improvements and extra features.

## How to Run the Project

1. Download or clone the GitHub repository.
2. Open the project folder.
3. Open `index.html` in a web browser.
4. The Task Tracker application will run directly in the browser.
5. No database or server setup is required because the application uses browser localStorage.

## Main Features

The Task Tracker application includes the following features:

- Create a new task
- View task list
- Edit existing task
- Delete task
- Assign task to a user
- Change task status
- Filter tasks by status
- Save task data using browser localStorage

## Agile Methodology

The team selected Scrum as the Agile SDLC methodology for this project. Scrum was chosen because it allows the project to be divided into short development cycles called sprints. This makes it easier to plan, implement, review, and improve the system step by step.

The project was completed using two short iterations.

### Iteration 1: Basic Task Tracker

The first iteration focused on creating a minimum viable version of the Task Tracker. The main goal was to build a working system with basic task management features.

Features delivered in Iteration 1:

- Add task
- View task list
- Delete task
- Save tasks using localStorage
- Basic user interface

Tag / milestone:

```text
v0.1-iteration1
```

### Iteration 2: Improved Task Tracker

The second iteration improved the system by adding new features and refining the user experience. This showed incremental development because the system became more complete after the first working version.

Features delivered in Iteration 2:

- Edit existing tasks
- Assign tasks to users
- Add task status: To Do, In Progress, Done
- Filter tasks by status
- Improve form validation
- Improve user interface

Tag / milestone:

```text
v0.2-iteration2
```

## Iterative and Incremental Development

This project applied iterative and incremental development by delivering the application in two stages.

In Iteration 1, the team developed the basic working version of the system. This version allowed users to create, view, delete, and save tasks.

In Iteration 2, the team improved the first version by adding editing, assigned users, status tracking, filtering, validation, and better UI design. This allowed the application to evolve from a simple task list into a more complete task management tool.

## GitHub Collaboration

GitHub was used to manage the project development process. The team used GitHub issues to plan tasks, branches to develop features separately, commits to record progress, and pull requests to review and merge changes.

The team followed this workflow:

```text
Issue → Branch → Commit → Pull Request → Review → Merge → Tag
```

## Branching Strategy

The project used the following branching strategy:

- `main`: Stable version of the application
- `feature/iteration1-basic-task-tracker`: Branch for Iteration 1 implementation
- `feature/iteration2-task-improvements`: Branch for Iteration 2 improvements
- `docs/update-readme`: Branch for README documentation update
- `docs/user-stories-test-plan`: Branch for user stories and test plan documentation

## GitHub Issues and Pull Requests

The project used GitHub Issues to divide work into clear tasks. Each task was linked to a branch and pull request.

Examples of GitHub tasks include:

- Build basic task creation and task list
- Improve task management with status, editing and filtering
- Update README with project details and run instructions
- Add user stories and test plan

Pull requests were used to review changes before merging them into the `main` branch.

The project has completed two iterations and includes the main required Task Tracker features. The GitHub repository also contains evidence of Agile planning, GitHub collaboration, pull requests, commits, and iteration milestones.
